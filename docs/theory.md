# Теория: Как работает Claude Code

## Что это

Claude Code -- CLI-инструмент от Anthropic для работы с кодом через терминал. Это не чат, а агент с доступом к файловой системе, git и shell.

## Как загружается сессия

```
claude (запуск в терминале)
  │
  ├── 1. Читает ~/.claude/settings.json
  │      Глобальные permissions: что запрещено, что спросить
  │
  ├── 2. Читает CLAUDE.md (в корне проекта)
  │      Контекст проекта, стиль кода, правила, запреты
  │
  ├── 3. Читает .claude/ (если есть)
  │      Проектные настройки, commands, skills, hooks
  │
  ├── 4. Сканирует структуру проекта
  │      Понимает что где лежит
  │
  └── 5. Создаёт СЕССИЮ
         Контекстное окно ~200K токенов
         Внутри: system prompt + правила + история диалога
```

## Контекстное окно -- главное ограничение

- Окно ~200K токенов -- "оперативная память" агента
- Всё что ты пишешь, файлы, результаты команд -- занимают место
- Когда окно заполняется -- агент "забывает" начало
- `/compact` -- сжимает контекст, оставляя суть
- `/clear` -- полная очистка (начать с нуля)
- Правило: длинная задача = разбивай на куски

## Что Claude Code может

- Читать и писать файлы
- Выполнять bash-команды (с учётом permissions)
- Работать с git (коммиты, ветки, PR)
- Запускать субагентов (параллельные задачи)
- Искать по проекту

## Permissions -- три уровня

| Уровень | Что делает | Пример |
|---------|-----------|--------|
| **allow** | Выполняет молча | Чтение файлов |
| **ask** | Спрашивает разрешения | `git push --force` |
| **deny** | Запрещено всегда | `rm -rf /`, `sudo` |

Без deny-правил агент может удалить файлы или сломать git.

## CLAUDE.md -- память проекта

Главный файл. Загружается в контекст при каждом запуске.

Содержит:
- Роль и контекст проекта
- Workflow-правила (Plan Mode, Self-Improvement, Verification)
- Стиль кода
- Пути и структуру проекта
- Git-правила
- Запреты

**Важно:** максимум 150 строк. Больше = агент хуже следует правилам.

## Hooks -- автоматизация действий

Hooks -- триггеры, которые срабатывают на события:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo 'Команда выполнена'",
            "timeout": 30
          }
        ]
      }
    ]
  }
}
```

Типы хуков:
- **PostToolUse** -- после выполнения команды/инструмента
- **PreToolUse** -- перед выполнением

Примеры использования:
- Автоматический lint после изменения файла
- Code review после коммита
- Уведомление после долгой операции

## Ключевые команды

| Команда | Что делает |
|---------|-----------|
| `/init` | Создаёт CLAUDE.md по проекту |
| `/compact` | Сжимает контекст |
| `/clear` | Полная очистка сессии |
| `Shift+Tab` | Plan Mode (только планирует) |
| "think hard" | Глубокий анализ |
| "ultrathink" | Максимально глубокий анализ |

## Субагенты

Субагенты -- параллельные процессы Claude:
- Один субагент = одна задача
- Держат основной контекст чистым
- Для ресёрча, тестирования, анализа
- Запуск: "Запусти субагента чтобы исследовать X"

## Как всё работает вместе

```
Ты пишешь задачу
    │
    ▼
Claude читает CLAUDE.md (правила)
    │
    ▼
Скилл GSD: "Сначала спецификация"
    │
    ▼
План в tasks/todo.md
    │
    ▼
Claude пишет код по правилам
    │
    ▼
Hooks: автопроверки
    │
    ▼
Claude Mem: сохраняет контекст
    │
    ▼
Следующая сессия: помнит что было
```
