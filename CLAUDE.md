## Workflow

### Plan Mode
- Для ЛЮБОЙ нетривиальной задачи (3+ шагов) -- сначала план
- Если пошло не так -- СТОП, перепланировка. Не продолжай вслепую
- Пиши план в tasks/todo.md с чекбоксами
- Используй субагентов для ресёрча (один субагент = одна задача)

### Self-Improvement Loop
- После КАЖДОЙ поправки от пользователя -- записать урок в tasks/lessons.md
- Написать правило, предотвращающее повторение ошибки
- Пересматривать уроки в начале сессии

### Verification Before Done
- Никогда не отмечай задачу готовой без proof
- Запусти тесты, проверь логи, покажи результат
- Спроси себя: "Одобрил бы это senior разработчик?"

### Autonomous Bug Fixing
- Получил баг -- чини сам, не спрашивай как
- Смотри логи, ошибки, падающие тесты -- и исправляй

## Task Management
1. План в tasks/todo.md с чекбоксами
2. Проверь план перед реализацией
3. Отмечай прогресс по ходу
4. Объясняй изменения на каждом шаге
5. Фиксируй уроки в tasks/lessons.md

## Core Principles
- Simplicity First -- минимальный impact, минимум кода
- No Laziness -- root cause, не костыли. Стандарты senior разработчика
- Minimal Impact -- трогай только нужное

## Стиль кода
- Python: snake_case, type hints, Google-style docstrings
- JS/TS: camelCase, JSDoc, strict TypeScript (no any)
- Максимальная длина строки: 100 символов
- Без magic numbers -- выносить в константы
- Каждая функция -- один уровень абстракции

## Git-правила
- Коммиты на русском: "Добавил авторизацию пользователей"
- Ветки: feature/название, fix/название, refactor/название
- НИКОГДА не push в main -- только через Pull Request
- Коммит после каждого завершённого куска работы
- Перед коммитом: запустить тесты

## Запрещено
- any в TypeScript, var в JavaScript
- Коммитить .env, секреты, API-ключи, токены
- sudo, rm -rf без явного подтверждения
- TODO без номера задачи
- Добавлять зависимости без обоснования
- Менять файлы, о которых не просили

## Обязательно
- Новый файл = новый тест
- Бэкап конфигов перед изменением
- Ошибки обрабатывать явно (no silent fails)
- Объяснять ЧТО и ПОЧЕМУ в каждом изменении
